# Automated News Briefing System
### Project Presentation

---

## What Was Built

An automated daily briefing system that:
- Logs into **4 German premium news sources** (Tagesspiegel Background, WirtschaftsWoche, SZ-Dossier, Capital)
- Scrapes today's articles including full content behind paywalls
- Generates an **AI summary** using a local LLM (Mistral-Small-24B)
- Produces a **PDF briefing** — delivered automatically every morning

**Platform:** n8n workflow automation on Kubernetes

---

## System Overview

```
Scheduler
   └─ Login & Scrape Articles     (Puppeteer / Chromium)
         └─ AI Summary             (vLLM — Mistral-Small-24B)
               └─ Generate PDF     (Pure Node.js)
                     └─ Deliver
```

**Infrastructure:**
- Kubernetes cluster (namespace: preme-n8n)
- Custom Docker image with Chromium + Puppeteer
- Corporate proxy for outbound traffic (Istio mesh)
- Credentials stored in Kubernetes Secrets

---

## Journey — Three Approaches Tried

### Attempt 1 — Selenium
- Standard browser automation tool
- **Problem:** Unreliable through corporate proxy, difficult session management
- **Result:** Abandoned

### Attempt 2 — TypeScript Custom Node (HTTP-based)
- Direct HTTP requests to news sites
- **Problem:** Paywall content is revealed by JavaScript in the browser — plain HTTP gets empty content
- **Result:** Abandoned

### Attempt 3 — Puppeteer inside n8n Code Node ✅
- Full browser automation running inside the workflow
- Reuses authenticated browser session across all article pages
- **Result:** Working

---

## Key Technical Challenges Solved

| Challenge | Solution |
|---|---|
| Istio blocking all outbound traffic | Route Chromium through corporate proxy |
| Puppeteer modules not found in runner subprocess | Dockerfile symlinks into default Node.js path |
| Cookie consent (shadow DOM) | Recursive shadow DOM traversal function |
| Login form rejecting keyboard input in external runner | Native `HTMLInputElement` value setter via `page.evaluate()` |
| Task runner hanging indefinitely | Replace `waitForFunction` with `setTimeout` |
| PDF blank in Linux container (no fonts) | Pure Node.js PDF with built-in Type1 fonts — no system fonts needed |

## PDF Output

Two PDF types generated:

**1. Raw Article Briefing**
- All scraped articles in one PDF
- Topic, title, author, date, teaser per article

**2. AI Summary Briefing**
- Executive summary generated by Mistral-Small-24B
- Bullet-point format with key topics highlighted
- Rendered from LLM markdown output (headings, bold labels)

Both PDFs use pure Node.js generation — no Chromium required, works in any container.

## Lessons Learned

1. **Paywall scraping requires a real browser** — HTTP clients cannot execute the subscription-check JavaScript that reveals content
2. **n8n internal task-runner has hidden constraints** — child subprocess does not inherit environment variables; `waitForFunction` causes deadlocks
3. **Minimal Linux containers have no fonts** — avoid Chromium for PDF rendering; built-in PDF fonts are more reliable
4. **Istio requires explicit proxy configuration** for browser-based outbound traffic
5. **Debugging in n8n runners** — `console.log` unreliable; use `throw new Error()` or return debug JSON
